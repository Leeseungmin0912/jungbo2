# Cyber Security Tool

Python 기반의 Windows 보안 점검 및 로그 분석 도구입니다.  
포트 스캔, 위험도 분석, 보안 로그 확인, 보고서 저장 기능을 통해 사용자가 자신의 시스템 상태를 직접 점검할 수 있도록 제작했습니다.

---

## 프로젝트 소개

Cyber Security Tool은 사용자가 자신의 PC 환경을 간단하게 점검하고,  
열려 있는 포트와 보안 로그를 확인하여 기본적인 위험 요소를 스스로 파악할 수 있도록 만든 프로그램입니다.

정보보안 분야를 공부하면서 단순한 이론 학습에 그치지 않고,  
직접 기능을 구현해보며 실습 경험을 쌓고 포트폴리오로 활용하기 위해 제작했습니다.

특히 이번 프로젝트에서는 단순 포트 스캐너 수준을 넘어서,  
Windows 보안 이벤트 로그를 분석하고 의심스러운 로그인 흔적을 확인할 수 있도록 기능을 확장했습니다.

---

## 제작 목적

- 사용자가 자신의 시스템 보안 상태를 직접 점검할 수 있는 도구를 만들기 위해
- 정보보안 분야에서 배우는 개념을 실제 프로그램으로 구현해보기 위해
- 포트 스캔, 리스크 점수 계산, 보안 로그 분석 기능을 직접 개발하며 실습 경험을 쌓기 위해
- 추후 포트폴리오와 자기소개서, 면접 자료로 활용할 수 있는 프로젝트를 만들기 위해

---

## 주요 기능

### 1. 포트 스캔
- 사용자가 입력한 IP 주소의 포트를 스캔
- 열려 있는 포트를 확인하고 포트 번호와 서비스명 출력
- 자주 사용되는 주요 포트 정보를 함께 표시

### 2. 위험 포트 기반 리스크 스코어 계산
- 열린 포트 중 보안상 주의가 필요한 포트에 가중치를 부여
- 현재 위험 점수를 계산하여 표시
- 단순히 포트 개수만 보는 것이 아니라, 어떤 포트가 열려 있는지에 따라 위험도를 판단할 수 있도록 구성

예시 위험 포트:
- 21 FTP
- 23 Telnet
- 445 SMB
- 3389 RDP
- 3306 MySQL
- 5432 PostgreSQL
- 8080 HTTP-Alt

### 3. IP 정보 확인
- 대상 IP의 국가, 도시, ISP 등 기본 정보 확인
- 외부 IP를 조회할 때 네트워크 정보와 함께 참고 가능

### 4. Windows 보안 로그 분석
- Windows Security 로그를 CSV 형태로 저장 및 분석
- 주요 이벤트 ID를 기반으로 로그인 흔적과 권한 상승 여부 확인
- 반복적으로 발생한 보안 이벤트를 요약해서 보여줄 수 있음

주요 분석 이벤트:
- **4624** : 로그인 성공
- **4672** : 관리자 권한이 포함된 로그인
- **5379** : 자격 증명 관련 정보 조회

### 5. 의심 로그 확인
- 특정 시간대에 로그인 관련 이벤트가 집중적으로 발생했는지 확인
- 사용자가 PC를 사용하지 않은 시간대의 로그를 보고 이상 여부를 판단할 수 있도록 보조
- 덮개 열림, 절전 복귀, 자동 로그인, 시스템 서비스 실행 등 정상 가능성도 함께 고려할 수 있도록 설계

### 6. 보고서 저장 기능
- 점검 결과를 텍스트 또는 보고서 형태로 저장 가능
- 필요 시 결과를 기록으로 남겨 포트폴리오나 점검 이력으로 활용 가능

### 7. GUI 기반 인터페이스
- Tkinter 기반의 간단한 GUI 제공
- 버튼 클릭으로 기능을 실행할 수 있어 초보자도 사용 가능
- 로그 창을 통해 진행 상황과 결과를 쉽게 확인 가능

## 사용 기술

- **Python**
- **Tkinter** : GUI 구성
- **socket** : 포트 스캔
- **requests** : IP 정보 조회 및 네트워크 요청
- **threading / concurrent.futures** : 병렬 처리
- **csv / os / datetime / re / hashlib** : 로그 처리 및 파일 저장 기능

---

## 프로그램 흐름

1. 사용자가 대상 IP와 포트 범위를 입력
2. 포트 스캔 실행
3. 열린 포트 및 서비스 정보 출력
4. 위험 포트를 기준으로 리스크 스코어 계산
5. 필요 시 IP 정보 확인
6. Windows 보안 로그를 CSV로 저장 후 분석
7. 이벤트 ID별 발생 횟수 및 시간대 확인
8. 결과를 보고서로 저장

---

## 기대 효과

- 사용자가 자신의 PC 환경에서 열려 있는 포트를 직접 확인할 수 있음
- 위험도가 높은 포트를 중심으로 기본 보안 점검 가능
- Windows 보안 로그를 통해 로그인 흔적과 이상 동작 여부를 확인할 수 있음
- 정보보안 학습 과정에서 실습형 프로젝트 경험을 쌓을 수 있음
- 단순한 예제가 아닌 실제 점검 흐름을 가진 프로젝트로 포트폴리오 활용 가능

## 프로젝트를 통해 배운 점

이 프로젝트를 진행하면서 단순히 포트를 스캔하는 기능만 구현하는 것이 아니라,  
실제로 보안 도구가 어떤 흐름으로 동작해야 하는지 고민하게 되었습니다.

특히 다음과 같은 부분을 직접 경험할 수 있었습니다.

- 네트워크 포트의 개념과 위험 포트의 의미
- 로그인 이벤트와 권한 관련 Windows 보안 로그 구조
- 단순한 로그 수집이 아니라, 해석과 판단이 중요하다는 점
- GUI 프로그램에서 사용자 편의성과 기능 구성을 함께 고려해야 한다는 점
- 실제 프로그램을 배포하고 관리하는 과정의 중요성

## 아쉬운 점 / 개선 예정

- 현재는 기본적인 포트 스캔과 이벤트 로그 분석 중심
- 보안 로그의 세부 항목(Logon Type, Source IP, Account Name)까지 자동 분류하는 기능은 추가 개선 가능
- 의심 이벤트 발생 시 자동 경고 및 상세 보고서 생성 기능 보완 예정
- 취약점 스캐너, 비밀번호 강도 검사기, 해시 생성기 등 추가 기능 확장 가능
- 향후 자동 업데이트 및 배포 구조도 더 안정적으로 개선 예정
